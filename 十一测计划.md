# 十一测计划

> 十一测命题：把 shrinkwrap 的 σ 从**固定线性衰减**（sigma0=3.0→sigma_end=1.0）改成 **Yoshida 2024 自适应 σ**——根据 support 面积变化幅度动态切（剧变时大 σ 强平滑、稳定后锁小 σ 紧贴）。**对症十测两个 support 失稳问题**：① 组4 unet_raar 在 1700 轮 support 剧变→相位发散突崩；② unet_raar/tanh_full 的"中心-背景"两种模式（R12，iou 低=背景斑驳）。

> 用户决策：**直接上自适应 σ（不扫 thresh_frac）+ 4 法矩阵 + 10 组**。理由：自适应 σ 改的是公共的 support 估计，影响所有方法，应用 4 法矩阵全面看效果（而非只 2 法）。RAAR 作稳定基线、unet_raar/tanh_full 看模式 B 锁定、**HIO 重启**看 support 改进能否缓解周期斜线（C14 之前归因为 HIO 负反馈震荡，但 support 估计或是另一因素，值得一试）。理论见 [调研报告.md](调研报告.md) §3.9、[Yoshida 2024](https://pmc.ncbi.nlm.nih.gov/articles/PMC10833425/)。

## 1. 背景与动机

### 1.1 十测留下的两个 support 失稳问题

- **组4 unet_raar 突崩**：1600 轮 iou≈0.48 / ssim 0.93 稳态 → 1700 轮 iou 0.08 / ssim 0.62，**support 剧变后相位发散**（C18）。固定 σ schedule 在 support 失稳时无力拉回。
- **两种模式**（R12）：unet_raar / tanh_full 落入"中心清晰+背景斑驳（iou 低）"或"中心模糊+背景干净（iou 高）"，support 估计质量（iou）是分野钥匙。

### 1.2 Yoshida 2024 的解法

[Yoshida 2024](https://pmc.ncbi.nlm.nih.gov/articles/PMC10833425/) 的 improved shrinkwrap：σ 不固定递减，而按 **support 面积变化幅度（|Δ OS ratio|）自适应**——剧变时大 σ=2 强平滑（抑制 support 突变导致 PR 发散）、稳定后切 σ=0.9 紧贴并锁定。论文实测显著提升 realistic map 命中率（§3.6/图3）。

### 1.3 命题

自适应 σ 能否**稳定 support 估计**，从而：
- 防 unet_raar 突崩（组4 式 support 剧变→发散）；
- 把 unet_raar/tanh_full 锁在"背景干净"模式（iou 高）；
- 缓解 HIO 周期斜线（若 support 估计是 C14 的因素之一）。

## 2. 自适应 σ 机理与项目实现

### 2.1 Yoshida 原版（论文 §3.6）

- 每 100 HIO 轮做一次 SW，记录 OS ratio = total_area / support_area（过采样比，>2）。
- **|Δ OS ratio| > 2**（support 面积剧变）→ 用大 ζ=2 强平滑；
- **|Δ OS ratio| ≤ 2**（稳定）→ 切小 ζ=0.9 紧贴，**一旦切到 0.9 即锁定**后续不再变。

### 2.2 项目实现（`utils.py` 新增 `AdaptiveSigma` 类）

```python
class AdaptiveSigma:
    """Yoshida 2024 自适应 σ（论文 §3.6 原版）：监控过采样比 OS ratio（total/support）
    变化幅度，剧变（|Δ|>2）用大 σ 平滑、稳定切小 σ 锁定。"""
    def __init__(self, sigma_large=3.0, sigma_small=1.0, delta_thresh=2.0):
        # sigma_large/small 用项目 sigma0/sigma_end（保与固定 schedule 同值域，唯一变量是 σ 决定逻辑）
        # delta_thresh=2.0 是 Yoshida 论文原值
        self.sigma_large, self.sigma_small, self.delta_thresh = sigma_large, sigma_small, delta_thresh
        self.prev_os, self.locked, self.sigma = None, False, sigma_large  # 首次用大 σ

    def next(self, support):  # support 是当前 mask 张量 [1,1,H,W]
        os_ratio = support.numel() / max(support.sum().item(), 1.0)  # OS ratio = total/support，>2
        if self.locked:
            return self.sigma_small
        if self.prev_os is not None:
            if abs(os_ratio - self.prev_os) > self.delta_thresh:  # 剧变→大 σ 平滑
                self.sigma = self.sigma_large
            else:                                                   # 稳定→小 σ 锁定
                self.sigma, self.locked = self.sigma_small, True
        self.prev_os = os_ratio
        return self.sigma
```

**忠实论文（A 版）**：OS ratio = total/support（经典过采样比，>2）、判据 |Δ|>2、剧变大 σ / 稳定小 σ 锁定——全部按 Yoshida §3.6 原版。**唯一适配**：σ 大小用项目 `sigma0=3.0/sigma_end=1.0`（不用论文的 2/0.9），保与固定 schedule 同值域可比，使"σ 决定逻辑"成为唯一变量。`delta_thresh=2` 在我们小物体场景对应 support 占比相对变化约 25–35%，足以识别组4 式暴跌（80%+），probe 时打印 OS ratio 序列确认。

### 2.3 参数选择文献依据（[调研报告.md](调研报告.md) §3.9）

- 项目 thresh_frac=0.15、sigma0=3.0 正是 Marchesini 2003 原始配置 ✓，**自适应 σ 模式下 thresh_frac 保持 0.15 不变**（唯一变量 = σ 决定方式）。
- σ≈3 pixel、threshold 0.1–0.2 均符合文献主流。

## 3. 目标与矩阵

**4 法 × 10 组 × 自适应 σ × 3000 轮 = 40 个实验**（十测固定 σ 5 组作历史对照，不重跑）。

| 维度 | 取值 | 说明 |
|---|---|---|
| 方法 | **HIO / tanh_full / RAAR / unet_raar** | 十测 4 法；自适应 σ 是公共改动，全法都试 |
| σ 模式 | **自适应（Yoshida）** | 唯一新变量；对照 = 十测固定 σ schedule（历史数据） |
| 轮次 | **3000（4 法统一）** | 自适应 σ 是主变量，轮次须对齐；RAAR 2000 已收敛，3000 无害 |
| 组数 | **10** | 取统计（十测 5 组下 unet_raar 模式命中率统计弱） |
| thresh_frac / β / γ | 0.15 / 0.7 / 0.8（同十测） | 不动，隔离 σ 变量 |

### 控制变量

- **十一测 vs 十测**：唯一差 = σ 决定方式（自适应 vs 固定 schedule），其余参数全相同 → 回答"自适应 σ 是否改善各法"。
- **4 法之间**：σ 模式、轮次、组数全对齐，唯一差 = 算法核 → 回答"自适应 σ 下哪种算法核最受益"。

### 核心看点（R8 看图 + R12 iou + C13 收敛曲线）

1. **unet_raar 突崩是否消失**（组4 式 support 剧变→发散）：10 组里还有几组突崩？收敛曲线是否丝滑到底。
2. **unet_raar / tanh_full 是否锁住模式 B**（iou 显著升高 + 背景干净），10 组模式命中率 vs 十测。
3. **HIO 周期斜线是否缓解**（C14）：若 support 改进能减少斜线，C14 归因需补"support 估计"因素。
4. **RAAR 是否依然最稳**（C15 基线）：自适应 σ 不应破坏 RAAR 稳定性。

## 7. 风险与权衡

1. **自适应 σ 未必救突崩**：组4 突崩根因可能是 UNet 软约束的相位发散（C18，loss 作用域深层），support 剧变只是表象。若自适应 σ 锁住了 support 但突崩仍发生，说明根因在 loss 不在 support → 转 §8 备选。
2. **HIO 重启风险**：HIO 之前暂停（C14），自适应 σ 未必能救（C14 主因是 HIO 负反馈震荡）。HIO 仍崩则记录"support 改进救不了 HIO 震荡"，印证 C14 归因，不强求。
3. **`rel_thresh` 是猜的**：0.15（参考 Yoshida 换算）未必最优。probe 时打印每次 SW 的 support_frac + σ 决策，确认切换逻辑合理；不合适则调。
4. **时长 5–7 小时**：40 个 3000 轮，长实验。后台跑 + 断点续跑兜底。
5. **自适应 σ 改变所有法的行为**：与十测固定 σ 不严格可比（σ 模式变了），但十一测内部 4 法公平，且十测 5 组历史数据仍可作趋势对照。

## 8. 备选方向（若自适应 σ 不奏效）

若 §7 风险1 命中（support 锁住但仍突崩），根因在 loss 作用域：

1. **全图 loss 加 support 内加权**：保护中心梯度不被 support 外分摊（直击 C17"得背景失中心"）。
2. **UNet loss 加 support 外 L2 惩罚**：显式压背景干净，防相位发散（C18）。
3. **扫 thresh_frac**：十一测原方案（§2.3 文献依据），作为自适应 σ 不奏效时的 fallback。
