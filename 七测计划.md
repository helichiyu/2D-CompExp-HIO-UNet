# 七测计划

> 本文档规划第七次测试。基于六测核心遗留（C12：三种方法都是概率恢复，单次结论非稳健），七测做**大规模重复取统计**：10 组 × 5 实验 = 50 个实验，HIO 5000 轮 / UNet 1500 轮。代码改造三处：① 每个实验（含 HIO 3 次）完整保存；② 多组循环 + 断点续跑；③ summary 汇总 + 跑完按类型推荐看图。

> 理论见 [调研报告.md](调研报告.md)，六测认知见 [六测汇报.md](六测汇报.md)、[实验猜想与结论.md](实验猜想与结论.md)，历史见 [初测汇报.md](初测汇报.md)、[二测汇报.md](二测汇报.md)、[四测汇报.md](四测汇报.md)、[五测汇报.md](五测汇报.md)、[六测汇报.md](六测汇报.md)。

---

## 1. 背景与动机

六测最重要的认知升级是 **C12：三种方法（HIO / sigmoid / tanh+HIO）的恢复都是概率性事件**，单次结果随机，方向性可靠但绝对数值非稳健。六测遗留明确（[实验猜想与结论.md](实验猜想与结论.md) §4）：**多次重复取命中率/均值±方差**才能得稳健结论。

另：用户要求**每次测试结果完整保存**（六测 HIO 跑 3 次但只存"代表"，不妥）。七测改造保存逻辑 + 大规模重复。

附带验证六测 §4.2 的张力：HIO 加轮次到 5000，看"中心轮廓 + 轮廓循环"中间态能否完全收敛（C10）。

---

## 2. 目标

1. 10 组独立重复 × 5 实验 = **50 个实验**，取统计（命中率/均值±方差）；
2. HIO 加轮次至 **5000**（验中间态能否完全收敛）；
3. **每个实验完整保存**（含 HIO 3 次，不只代表）；
4. 跑完按类型推荐 top 组供看图，写七测汇报，把 C12 升级为统计性结论。

---

## 3. 实验矩阵

每组 5 个实验位（沿用六测）× 10 组：

| 位 | 方法 | HM | 轮次 | support 外 |
|---|---|---|---|---|
| HIO ×3 | 传统 HIO（relaxed γ=0.9） | 开 | 5000 × 3 | relaxed HIO |
| tanh+HIO | UNet tanh + HIO 反馈 | 开 | 1500 | relaxed HIO |
| sigmoid | UNet sigmoid + 置 0 | 开 | 1500 | 置 0 |

> 10 组的差异**完全来自随机性**（phase seed 不固定、CUDA 非确定性 C6）——即 10 次独立重复，正是取统计所需的样本。不再把随机性当噪声，而当统计对象。

---

## 4. 代码改造（main.py）

### 4.1 完整保存（用户第一点）
- 实验1·HIO 3 次都**各自 dump_experiment**（`hio_1/`、`hio_2/`、`hio_3/`），不只存代表；
- 50 个实验每个独立全套：`real_space/spectra/support/convergence.png` + `metrics.csv` + `history.csv` + `state.pt`。

### 4.2 多组循环 + summary 汇总
- 外层 `N_GROUPS=10` 循环，每组跑 5 个实验；
- 每组出一个 `comparison_g<组号>.png/csv`（该组 HIO 取末轮 ssim 最高者作代表）+ 该组 HIO 3 次并排图 `hio_3runs.png`；
- 全部完成后扫所有 `group*/` 的 `metrics.csv`，汇总成 `summary.csv`（50 行：组号、类型、6 指标）。

### 4.3 断点续跑（用户确认）
- 命令行加第 3 个参数：`main.py [HIO_ITER] [UNET_ITER] [RUN_DIR]`。传 `RUN_DIR` 则续跑该目录，不传则新建 `run_<时间戳>`；
- `progress.json` 记录 `{done_groups, n_groups, hio_iter, unet_iter}`。续跑时：跳过 `done_groups`，校验 iter 配置一致（不一致报警终止）；
- **每组完成即落盘 + 更新 progress.json**（组粒度，中断最多损失当前组 ~31 min）。

### 4.4 目录结构
```
results/run_<时间戳>/
├── config.txt              # 本次配置（轮次、组数、超参）
├── progress.json           # 断点续跑进度
├── group01/
│   ├── hio_1/ hio_2/ hio_3/    # 各自全套 + hio_3runs.png
│   ├── tanh_hio/  sigmoid/     # 各自全套
│   └── comparison_g01.png/csv
├── group02/ ...
├── ...
├── group10/ ...
└── summary.csv             # 50 实验末轮指标汇总
```

### 4.5 新 main() 流程（骨架）
```python
def main(run_dir=None, hio_iter=5000, unet_iter=1500, n_groups=10):
    run_dir = run_dir or make_run_dir()
    prog = load_or_init_progress(run_dir, hio_iter, unet_iter, n_groups)  # 续跑校验
    rho_work, amp_orig, rho_init, ref_edges, support_gt, ... = preprocess()  # 共享（控制变量）
    for g in range(1, n_groups + 1):
        if g in prog['done_groups']:
            continue                       # 断点续跑跳过
        group_dir = os.path.join(run_dir, f'group{g:02d}')
        # HIO ×3（各 5000 轮，不同 phase seed）→ 各自 dump hio_1/2/3
        # tanh+HIO（1500 轮）→ dump tanh_hio/
        # sigmoid（1500 轮）→ dump sigmoid/
        # 该组 comparison_gXx + hio_3runs.png
        prog['done_groups'].append(g); save_progress(run_dir, prog)
    write_summary_csv(run_dir, n_groups)   # 扫 50 实验 metrics 汇总
```

## 7. 注意事项

1. **耗时约 5.2 小时**：后台跑；断点续跑防中断（组粒度，中断最多损失 ~31 min）。
2. **磁盘 200–300 MB**：50 实验 × 全套（state.pt 占大头）。
3. **随机性即统计对象**：10 组的差异正是 C12 要的样本，不再当噪声。
4. **HIO 5000 轮**：验六测中间态能否完全收敛（C10 张力），也提供更多 HIO 样本看命中率。
5. **看图为准（R8）**：推荐仅按指标初筛，HIO 场景 ssim 失真，最终你看图定。
