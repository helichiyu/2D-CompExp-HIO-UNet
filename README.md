# 对比实验：传统 HIO vs 未训练 UNet 相位恢复

在同一张图（567.png）上对比两种二维相位恢复方法，用**控制变量法**验证"像空间处理方式"对恢复效果的影响：

- **甲方 = 传统严格 HIO 算法**（像空间：硬替换振幅，保留新相位）
- **乙方 = 未训练 UNet**（像空间：UNet 输出做 FFT，振幅算 MSE 反传更新网络）

双方**实空间约束完全一致**（正值 + 实数 + 直方图匹配 + shrinkwrap 支撑域），唯一变量是像空间处理。

---

## 背景

X 射线晶体学 / 相干衍射成像中，探测器只能测得傅里叶振幅 |F|，**相位丢失**（相位问题）。相位恢复要从 |F| 反推能重建物体的相位。本实验从 567.png 的傅里叶振幅出发，丢掉相位，分别用 HIO 和未训练 UNet 恢复。

## 测试建议（重要）

UNet 完整迭代（5000 轮）耗时长（GPU 约 30 分钟，CPU 更久）。**强烈建议先用少轮次验证能跑通，再缓慢增加轮次**，避免一上来就跑完整轮次、等很久才发现问题：

1. **先跑 `smoke_test.py`**（基础函数 + HIO 100 轮 + UNet 50 轮，几分钟）验证环境和数据流能跑通：
   ```bash
   D:\anaconda3\envs\use\python.exe smoke_test.py
   ```
2. **确认跑通后**，缓慢增加轮次观察收敛趋势——临时改 `main.py` 顶部的 `UNET_ITER` / `HIO_ITER`：
   - 500 轮 → 看是否开始收敛
   - 1000–2000 轮 → 看趋势
   - 最后再跑完整 5000 轮出最终结果
3. 调参时同理——先少轮次验证参数方向对，再放量。

> 相位恢复 / 未训练网络对超参敏感，少轮次试探能大幅节省时间。`smoke_test.py` 就是为此设计的短迭代验证脚本。

## 关键设计

1. **极性翻转（暗背景表示）**：567.png 是白背景+黑物体，直接归一化会让实空间约束方向反掉。内部统一用 `ρ_work = bg_val − img`（物体亮、背景≈0）。数学上 `FFT(c−f)` 与 `FFT(f)` 的非直流振幅完全相同，故**不影响振幅约束，控制变量不破**。

2. **保留直流**：相位恢复需要平均密度信息，去直流会导致迭代不收敛（初测第一个修复）。

3. **严格 HIO**（β=0.7）：support 内且 ρ′≥0 → 保留 ρ′_k；否则 ρ_k − β·ρ′_k。

4. **shrinkwrap 动态支撑域**：高斯模糊 + 阈值二值化估物体区域，不偷看原图。`run_hio` 支持 `warmup`（前 N 轮固定宽松 support 让 ρ_k 成形）+ `init_support` + `fixed_support`（调试用）。

5. **直方图匹配**（Zhang-Main 300 等数量 bin，a·ρ+b 线性映射）：参考直方图从原图统计，只在 support 内做。`hm_start` 控制开启时机。

6. **评估前投影**：HIO 输出的 support 外是反馈值（可能为负），评估前先 `clamp_min(0) × support_pred` 投影到可行域，甲乙双方一致。

7. **数值约定**：图像全程 [0,1]；振幅用 `amp/amp.max()` 归一化算 loss（让 loss 尺度合理）。

## 评估指标（全套）

| 指标 | 含义 |
|---|---|
| PSNR | 峰值信噪比（图像域，[0,1]） |
| SSIM | 结构相似性（11×11 高斯窗口，自实现） |
| Pearson CC | 线性相关系数 |
| amp_cc | 振幅域相关系数 ΣF_obs·F_cal·cosΔφ / √(ΣF_obs²·ΣF_cal²) |
| Δφ | 平均相位误差（arccos(cos) 防 2π 卷绕） |
| support IoU | 恢复支撑域与真值支撑域的交并比 |

## 初测状态（已知问题）

**HIO 在不偷看原图（动态 shrinkwrap）时目前不收敛**——根因是 support 估计的"鸡生蛋"循环依赖（详见 [初测汇报.md](初测汇报.md)）：

- HIO 实现本身正确（用真值 support 时 ssim 0.95 / amp_cc 0.93 / IoU 0.77 漂亮收敛）；
- 但 567.png 是细线条分子结构，对 support 紧度极苛刻，不偷看时估不出准确 support；
- **这本身是实验价值**：HIO 强依赖准确 support，正是 UNet 方法的存在理由。下一步跑 UNet 完整迭代，验证它能否脱 support 收敛。

`main.py` 默认配置跑出来的 HIO 结果会偏差（动态 shrinkwrap 模式）；如需 HIO 理想基线，可在 `main.py` 里给 `run_hio` 传 `fixed_support`（从原图估的真值 support）。
