# 八测计划

> 本文档规划第八次测试。基于七测 50 实验统计的核心遗留 **C14（HIO 周期斜线 = 负反馈震荡；wrapping 次要）**，八测按"风险递增、先吃现成果子"的顺序推进：① 多次重建对齐平均 → ② PRTF 客观评估 → ③ tanh+HIO 震荡深挖 → ④ 阶段汇总 → ⑤ ER+HIO 交替（核心，先单跑验证）。

> 前三步**不动迭代算法**（离线评估 + 公式微调），⑤ 才动 HIO 核心。每步都有独立产出，跑完一步可单独写小结。

> 理论见 [调研报告.md](调研报告.md)，七测认知见 [七测汇报.md](七测汇报.md)、[实验猜想与结论.md](实验猜想与结论.md)。

## 1. 背景与动机

七测把"三种方法概率恢复"升级为带命中率的统计性结论（C12），并定位了周期斜线机理（C14）：主因 = HIO 负反馈震荡/stagnation，次因（存疑）= FFT 卷绕。七测遗留的八测方向经筛选后（撤回"加大 padding"——padding 充足、wrapping 非主因；证伪"HIO 加轮次"——5000×30 都没收敛）收敛到 4 条，本计划按风险递增排序推进。

关键约束：七测 50 实验的 `state.pt`（`best_rho` 已配准）现成可用——阶段①②直接离线复用，不重跑迭代。

## 3. 阶段 ①② 详细（先做，合一离线脚本）

### 3.1 数据来源
七测 `results/run_20260705_000538/`，扫所有 `groupNN/{hio_1,hio_2,hio_3,tanh_hio,sigmoid}/state.pt`。每个 `state.pt = {best_rho, best_rho_raw, hist}`，其中 **`best_rho` 已 `register_to_gt` 配准**（平移/共轭反转/符号三歧义已消除），可直接平均/算 PRTF，无需重配准。

三类样本：
- **HIO**：30 个（10 组 × hio_1/2/3）
- **tanh+HIO**：10 个（10 组 × tanh_hio）
- **sigmoid**：10 个（10 组 × sigmoid）；额外按 `summary.csv` 的 ssim 阈值（>0.8）自动挑"相变子组"单独算（预期 g6/g8）。

### 3.2 阶段①：实空间平均
- 每个重建先 `clamp_min(0)`（HIO/tanh 末轮有负反馈值；sigmoid 已 [0,1] 无影响），再同类栈平均。
- 平均后 `evaluate_all(align_to_gt=False)`（已配准，免重复配准）算 6 指标。
- 出图：原图 / 该类平均 / 绝对误差（沿用 `plot_real_space` 风格）。
- **预期**：HIO 30 张斜线倾角/周期随机 → 平均后斜线相互抵消、露出中间物体（C14 的直接验证）；sigmoid 全 10 个平均会被 8 张未相变稀释（故加相变子组）。

### 3.3 阶段②：PRTF（Phase Retrieval Transfer Function）
- 定义（Latychevskaia §3.5）：对 N 次独立重建 ρ_n（已配准），复傅里叶系数 F_n = FFT(ρ_n)，
  $$\mathrm{PRTF}(k) = \frac{|\langle F_n(k)\rangle|}{\langle|F_n(k)|\rangle}$$
  ∈[0,1]，衡量该频率多次重建的**相位一致性**。1=完全一致（收敛），0=相位随机。
- 用 `best_rho`（已配准、**不 clamp**——clamp 是非线性、会引入伪相位）算。
- 径向平均：`fftshift` 后按到 DC 的像素半径分桶，得 1D 曲线。
- 分辨率：PRTF 跌到 **1/e≈0.368** 与 **0.5** 的频率，报对应像素周期（无真实像素尺寸，暂报归一化频率/像素）。
- 出图：三类 PRTF 曲线（横轴频率像素、纵轴 PRTF，标阈值线）。
- **预期**：HIO PRTF 极低（相位高度不一致，证 C14）；sigmoid 相变子组 PRTF 高（相位一致，证 C13）。

### 3.4 脚本：`offline_eval.py`
```
python offline_eval.py <run_dir> [n_groups]
```
- 默认跑全量（avg + prtf），输出到 `<run_dir>/offline/`：
  - `avg_<type>.png` / `avg_metrics.csv`（阶段①）
  - `prtf_<type>.png` / `prtf_resolution.csv`（阶段②）
  - `prtf_curves.csv`（1D 曲线数据，供离线复算）
- 不动 `hio.py`/`unet_pr.py`/`main.py`。复用 `utils.register_to_gt`/`evaluate_all`/`fft_amp_phase`。

## 4. 阶段 ③④⑤ 方向（先做①②，再细化）

### 4.1 阶段③：tanh+HIO 震荡深挖（C8 延续）
七测 tanh+HIO 末轮 PSNR 漂移 −11.8~+14.3、曲线震荡。当前 `γ=0.9, β=0.7`。扫 `γ∈{0.7,0.8,0.9,0.95} × β∈{0.5,0.7,0.9}`，每组跑短轮次（如 1500），看 PSNR 漂移幅度与曲线震荡。`unet_pr.py` 的 γ/β 已参数化，加一个扫描入口脚本即可。

### 4.2 阶段④：阶段汇总
写八测阶段汇报（①②③ 结果），把"平均能否抑斜线""PRTF 是否区分收敛"沉淀为结论；若涉及新机理，查文献并更新 `调研报告.md`。

### 4.3 阶段⑤：ER+HIO 交替（核心，谨慎）
- **ER（Error Reduction）**：support 内取 `clamp_min(0, ρ')`、support 外置 0。比 HIO 简单、不震荡但易陷局部最优。
- **ER+HIO 交替**（Latychevskaia §3.4）：几轮 ER + 几轮 HIO 交替，文献称可避 HIO stagnation/周期斜线。
- **先单跑验证**：在 `hio.py` 加 `mode='er_hio'`（参数化 ER/HIO 交替周期），写最小入口**只跑 ER+HIO 一条路径**（不进 main 矩阵），跑几千轮交用户看图反馈。
- **可行后才并入矩阵**：改 `main.py run_one_group` 加实验位。

## 7. 注意事项

1. **阶段①②不动算法**：纯离线读 state.pt，可反复跑、零迭代成本。先吃透这步再决定后面。
2. **PRTF 要求重建配准**：用 `best_rho`（dump 时已 `register_to_gt`），不要用 `best_rho_raw`。
3. **avg 用 clamp_min(0)**：评估态平均（HIO 负反馈值不抵消物体）；**PRTF 不 clamp**（保傅里叶线性）。
4. **sigmoid 子组**：相变是低概率（20%），全 10 个平均/PRTF 会被稀释，必须单独看相变组。
5. **HIO 平均的物理意义**：斜线倾角随机 → 平均抵消；若平均后物体清晰，是"C14 斜线为随机震荡"的直接可视化证据。
