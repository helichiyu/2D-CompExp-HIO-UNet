# 十测汇报

> 十测命题：把 RAAR（C15，干净背景）与 UNet（C16，中心质量）的优势结合——保 RAAR 反射骨架不变，把 P_M 从"硬替换振幅"换成 UNet 软约束（unet_raar），看能否兼得"中心清 + 背景净"。附带 tanh_full（全图 loss）验 C16 背景斑杂根源、γ=0.8 折中。
>
> **最强发现：unet_raar 命题失败（新增 C18）。** UNet 软约束替代 P_M 引入不稳定性——5 组 ssim 0.619–0.938 大波动（vs RAAR 5/5 稳定 0.946–0.952），组4 在 1700 轮 ssim 0.926→0.640 突崩（相位发散，loss 仍在降）。4/5 组中心质量（psnr）也未超 RAAR。**RAAR 反射框架对 P_M 的"精确性"敏感，近似投影（UNet）不能替代硬替换 P_M。** C16 归因成立（新增 C17）：loss 作用域是背景斑杂主因。γ=0.8 折中失败（HIO 5/5 崩）。
>
> 产物：`results/run_20260705_183557/`（5 组 × 4 法 + 每组 comparison + summary.csv）。计划见 [十测计划.md](十测计划.md)，认知演变见 [实验猜想与结论.md](实验猜想与结论.md)。

> ⚠️ **看图说明（R8/R9）**：本次视觉分析工具不可用（MCP 400），曲线层面的结论（组4 突崩、RAAR 稳定）基于收敛历史（C13 硬证据）。**用户看图补充了关键观察（§3.1）**：unet_raar / tanh_full 都存在两种收敛模式——"中心清晰+背景斑驳" ↔ "中心模糊+背景干净"，且 **iou 是区分这两种模式的关键指标**（iou 越高背景越干净）。这一观察重新解释了 5 组 ssim 的大波动，并指向十一测的 support 参数方向。

## 3. 收敛曲线证据（C13：丝滑度=真收敛信号）

> 这是本次最硬的证据（不依赖看图）。

**组1 RAAR（基线，真收敛）**：ssim 0.60→0.934（300 轮到位）→ 平台 0.946 丝滑无震荡；amp_residual 2.21→0.21 收敛。

**组1 unet_raar（天花板低）**：ssim 缓慢爬升，1900 轮仅 0.81 平台——曲线相对丝滑，但**收敛天花板显著低于 RAAR**（0.81 vs 0.946）。loss 持续下降到 8.6e-7，说明 UNet 在振幅域拟合良好，但实空间结构停在次优态。

**组4 unet_raar（相位发散，头号风险命中）**：
- 0–1600 轮：ssim 稳定在 0.93（看似收敛）
- **1700 轮：ssim 0.926 → 0.640 暴跌，psnr 19.2 → 10.2，iou 0.48 → 0.08**
- 关键：**loss 仍在下降（8.8e-7），FFT 振幅仍匹配 amp_orig，但实空间相位发散**
- 机理：UNet 软约束只显式约束振幅、不约束相位，网络跳到"振幅匹配但相位错误"的解——正是 [十测计划.md](十测计划.md) §3.1 末尾点出的本质风险，**§7 风险1（UNet 近似 P_M 破坏 RAAR 收敛证明）真实命中**。

**组5 unet_raar（唯一追平）**：ssim 0.94 丝滑收敛、iou 0.54（反超 RAAR 的 0.40）。证明 unet_raar **能**收敛到高质量，但不稳定（5 组仅 1 组）。

### 3.1 用户看图补充：两种收敛模式 + iou 是背景干净度的钥匙（R8）

用户看图发现 **unet_raar / tanh_full 都存在两种收敛模式**：
- **模式 A**：中心物体清晰，但背景斑驳
- **模式 B**：中心模糊，但背景干净

这解释了 §2.1 中 unet_raar（0.619–0.938）、tanh_full（0.471–0.929）的 ssim 大波动——不同组落入不同模式。**关键钥匙是 support_iou**：

| 模式 | iou 特征 | unet_raar 印证 | tanh_full 印证 |
|---|---|---|---|
| 背景干净（B） | iou 高 | 组5 iou 0.541 / ssim 0.938 | 组2 iou 0.487 / ssim 0.929；组4 iou 0.428 / ssim 0.901 |
| 背景斑驳（A） | iou 低 | 组1 iou 0.269；组3 iou 0.185 | 组3 iou 0.090 / ssim 0.471；组5 iou 0.086 |

**iou = shrinkwrap 估的 support 与真值 support 的交并比**。iou 高说明物体轮廓被正确圈出、support 外被正确压零 → 背景干净；iou 低说明 support 估计错乱 → support 外残留杂项（背景斑驳）。RAAR 5/5 iou 0.354–0.454（稳定中等偏高）正是其背景干净的来源（C15）。**iou 是比 ssim/psnr 更直接的"背景干净度"指标**，下一步应作为主监控量。

**组4 unet_raar 是两种模式之外的第三种——不稳定**：先收敛到高质量稳态（1600 轮 iou≈0.48 / ssim 0.93，属"背景干净"模式），1700 轮突崩（iou 0.08 / ssim 0.62），support 直接垮掉。这进一步印证 C18 的不稳定性。

**用户猜想（待十一测验证）**：两种模式可能由 loss 作用域（全图 loss）造成——全图 loss 让 support 内外梯度分摊，网络在"侧重中心"与"侧重背景"间漂移。**而 support 获取参数（shrinkwrap 紧致度/收缩节奏）可能是控制这两种模式的开关** → 十一测扫 support 参数。

## 4. 核心认知（详见 [实验猜想与结论.md](实验猜想与结论.md)）

### C18（新，十测）. unet_raar 命题失败——UNet 软约束替代 P_M 引入不稳定性、中心质量未超 RAAR

5 组 unet_raar ssim 0.619–0.938（std≈0.115）vs RAAR 5/5 稳定 0.946–0.952（std≈0.002）。组4 在 1700 轮 ssim 0.926→0.640 突崩（loss 仍降，相位发散）。4/5 组 psnr 低于或近 RAAR，仅组5 追平。

**机理**：硬替换 P_M 精确保留输入 r_s 的相位、只换振幅（非扩张投影，RAAR 收敛证明依赖此精确性）；UNet 软约束只显式约束振幅，**相位靠网络先验 + 反射迭代动态演化**，无收敛保证，会跳到振幅匹配但相位错误的解。**RAAR 反射框架对 P_M 精确性敏感，近似投影（UNet）不能替代硬替换 P_M。** §8 单跑的"unet_raar 中心更清晰"印象是单样本，5 组统计不支持中心质量提升（建议看图复核）。

### C17（新，十测）. loss 作用域是 C16 背景斑杂的主因

tanh_full（全图 loss）背景比九测 tanh+HIO（support 内 loss）干净——§8 单跑已由用户看图确认（R8）。机理（[十测计划.md](十测计划.md) §1.3）：support 内 loss `fft(raw*support)` → 链式法则下 support 外梯度恒为 0 → 网络在 support 外自由发挥 → 背景斑杂（C16）；全图 loss `fft(raw)` 给 support 外补梯度，背景改善。**代价**：全图 loss 分摊 support 内中心梯度 → 中心质量下降（tanh_full psnr 8.6–11.7，风险3命中）。**C16 归因到 loss 作用域成立**，但全图 loss 有"得背景失中心"的权衡。

### C14 细化（十测）. γ=0.8 折中失败；γ 越低越难收敛；暂停纯 HIO

HIO γ=0.8 5/5 ssim 0.215–0.292，全面崩溃，与九测 γ=0.7（ssim 0.21–0.32）未见改善。**用户看图补充（R8）**：γ 调低后 HIO 再没收敛过——中心不出清晰像、背景还繁杂。即 **γ 越低越难收敛**（γ 调小本是为抑制周期斜线，但代价是收敛能力下降），γ 在 0.7/0.8/0.9 之间无解。

**周期斜线（C14）根本原因仍未明**——γ 调低只是压制现象、并非根治，调低了也仍会出现。**在找到周期斜线的根本性原因之前，暂停尝试纯 HIO**（已被 RAAR 全面碾压，C15）。

## 5. 后续方向（十一测，用户决策）

用户判断：unet_raar / tanh_full 的"中心-背景"两种模式（§3.1）是下一步主攻——**调 support（轮廓）获取参数，把网络推向"背景干净"模式，再靠长轮次恢复中心**。具体：

1. **主攻 support 获取参数**：扫 shrinkwrap 的紧致度/收缩节奏（`sigma0`/`sigma_end`/`sigma_interval`/`sigma_total`/`thresh_frac`），找把 iou 推高、锁住"背景干净"模式的参数组合。**iou 作为主监控量**（§3.1）。
2. **只跑 unet_raar + tanh_full**（UNet 路线，有改善空间）；RAAR 已是稳定基线（C15），用九测/十测历史数据作对照、不必重跑；HIO 暂停（C14 根本原因未明）。
3. **拉长轮次到 3000**：十测 2000 轮 unet_raar 多数未收敛透（仍落在"背景斑驳"模式），3000 轮看能否让更多组进入"背景干净"模式 + 中心恢复。
4. **多跑几组**：十测 5 组下两种模式的分布统计性弱，加组数取模式命中率。
5. **备选**：若调 support 参数仍锁不住"背景干净"模式，回考 §3.1 猜想——给全图 loss 加 support 内加权（保护中心梯度）、或 UNet loss 加 support 先验项防相位发散（组4 崩）。

详见 [十一测计划.md](十一测计划.md)。
