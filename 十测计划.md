# 十测计划

> 本文档规划第十次测试。九测拿到 **RAAR 反射框架破 C14（C15）**——5/5 稳定 ssim 0.95、背景干净，项目首次拿到不靠概率的稳定高质量恢复；但 RAAR 中心物体不如 UNet，UNet 两路（tanh/sigmoid）中心好但背景斑杂（C16）。十测命题 = **把两者的优势结合**：保 RAAR 反射骨架不变，把其中 P_M 的"硬替换振幅"换成 UNet 软约束——UNet 回归它作为"约束振幅的手段"的本职，RAAR 的反射框架（背景干净的来源）完整保留。

> 十测还带出一个新认知（用户洞察）：**原 UNet 路线 loss 只在 support 内（`fft(raw*support)`）→ support 外无梯度 → 网络在 support 外自由发挥 → C16 背景斑杂**。十测组2 把 loss 改成全图直接验证这个猜想，unet_raar 的全图 loss 也从结构上规避此问题。

> 理论见 [调研报告.md](调研报告.md) §2.3/§2.5，九测认知见 [九测汇报.md](九测汇报.md)、[实验猜想与结论.md](实验猜想与结论.md)。

## 1. 背景与动机

### 1.1 九测留下的互补矛盾

- **RAAR（C15）**：反射框架 5/5 稳定 ssim 0.945–0.951、曲线丝滑、**背景干净无杂项**。反射无累加项 → 避 HIO 负反馈震荡。但**中心物体恢复质量不如 UNet**。
- **UNet 两路（C16）**：tanh/sigmoid **中心物体质量高**（部分图超 RAAR），但**背景斑杂、杂项多**。

两者优势互补：RAAR 胜在背景干净、UNet 胜在中心质量。十测主命题 = 把二者结合。

### 1.2 结合思路：RAAR 骨架不变，P_M 换 UNet 软约束

UNet 在本项目里的本职是**约束振幅的手段**（像空间振幅 MSE 反传）。纯 RAAR 的 P_M 是"硬替换振幅"（`IFFT(amp_orig, phase(x))`）。自然融合 = **保 RAAR 反射骨架一字不改，只把 P_M 这一个算子从硬替换换成 UNet 软约束**：

- RAAR 反射框架（`r_s → r_m → x_next` + β 松弛）完整保留 → 背景干净、不震荡的来源不变；
- P_M 从硬替换 → `UNet(r_s)` 软约束振幅 → UNet 平滑先验顺带把中心画清晰（中心质量的来源）；
- 控制变量最干净：纯 RAAR vs unet_raar，唯一差 = P_M 的实现（硬替换 vs UNet 软约束）。

### 1.3 loss 作用域洞察（C16 根源猜想）

精确拆解原 `run_unet` 的 loss（[unet_pr.py:149](unet_pr.py#L149)）：`rho_c = raw * support; loss = mse(|FFT(rho_c)|, amp_orig)`。链式法则下 `∂loss/∂raw = ∂loss/∂rho_c × support`，**support 外 support=0 → loss 对 support 外 raw 的梯度恒为 0**。即：

- UNet 在 support 外**完全没有梯度监督**，support 外输出是网络自由发挥；
- tanh+HIO 模式再把这些无监督的 support 外 raw 塞进 HIO 反馈 `γ·input − β·raw` → 杂项注入 `rho_next` → 背景自我循环累积 = C16。

十测组2 把 loss 改成**全图**（`fft(raw)` 不乘 support），给 support 外补上梯度，直接验证此猜想。unet_raar 的全图 loss 也从结构上规避（support 只在 `P_S` 单点作用，loss 全图不偏科）。

### 1.4 附带：γ=0.8 折中

九测 γ=0.7 抑制 HIO 周期斜线（视觉）但治不了震荡（指标跳）；七测 γ=0.9 出斜线。十测组1/组2 试 **γ=0.8** 折中。

## 3. 算法核心

### 3.1 unet_raar 迭代核（主菜）

RAAR 反射骨架（[raar.py:67-70](raar.py#L67-L70)）一字不改，只把 P_M 从硬替换换成 UNet 软约束：

$$x_{k+1} = \tfrac{\beta}{2}\big(R_M(R_S(x_k)) + x_k\big) + (1-\beta)\,R_S(x_k), \qquad R=2P-I$$

- `R_S(x) = 2·P_S(x) − x`：P_S 硬投影（support 外=0、内 clamp_min(0)），**同纯 RAAR，不变**。
- `R_M(r_s) = 2·P_M(r_s) − r_s`：P_M 从硬替换换成 UNet 软约束（**唯一改动**）。
- β=0.7（同纯 RAAR）。

伪代码（`run_unet_raar`）：

```python
for it in range(max_iter):
    # ① 实空间反射 R_S（no_grad，与纯 RAAR 逐字一致）
    with no_grad:
        r_s = 2 * proj_S(x, support) - x              # P_S 清 support 外 + clamp 正

    # ② P_M 换 UNet 软约束（带梯度训练，唯一改动）
    optimizer.zero_grad()
    raw = model(r_s.detach())                         # UNet 吃 r_s，tanh 输出
    amp_pred = fft_amp_phase(raw)[0] / amp_max        # 全图振幅（对齐 P_M 全图 FFT 语义）
    loss = mse(amp_pred, amp_orig_norm)
    loss.backward(); optimizer.step()                 # 训 UNet 成软振幅投影器

    # ③ 反射组合（no_grad，与纯 RAAR 逐字一致）
    with no_grad:
        proj_M = raw.detach()                         # UNet 输出顶替 P_M(r_s)
        r_m = 2 * proj_M - r_s                        # R_M = 2·P_M − r_s
        x_next = (beta/2)*(r_m + x) + (1-beta)*r_s    # β 松弛平均
        x_next = histogram_match(x_next, support, ref_edges)
        if it >= warmup and it % sigma_interval == 0: # 动态 support，沿用 RAAR 节奏
            support = shrinkwrap_support(x_next, sigma)
        x = x_next.detach()
```

**raw 的双重角色**：既是 UNet 输出（被振幅 loss 训练，平滑先验带中心结构），又是 P_M 的软投影结果（detach，纯数值）。梯度只走 loss 那条线，反射链是纯数值。

**硬替换 vs UNet 软约束的本质区别**（不只是"硬 vs 软"）：硬替换 P_M 显式保留输入 r_s 的相位、只换振幅；UNet 软约束只显式约束振幅，**相位靠网络先验 + 反射迭代动态演化**。这是否能补回合理相位，正是十测要回答的。

### 3.2 loss 作用域改动（组2）

`run_unet` 加参数 `loss_scope`：
- `'support'`（默认，原样）：`rho_c = raw * support; loss = mse(|FFT(rho_c)|, amp_orig)` —— support 内。
- `'full'`（组2 新增）：`loss = mse(|FFT(raw)|, amp_orig)` —— 全图，不乘 support。

迭代核其余不变。组2 调 `run_unet(..., loss_scope='full')`。

## 7. 风险与注意事项

1. **unet_raar 头号风险：UNet 是近似 P_M，破坏 RAAR 收敛性**。硬替换精确满足振幅=amp_orig，UNet 软约束只逼近；RAAR 收敛证明依赖 P_M 精确（非扩张），换近似投影后理论不保证收敛。**对策**：步骤 2 先单跑看图，不收敛则调 β（反射强度）、UNet 输出层、输入归一化；最坏情况退回纯 RAAR（已证 C15 成立）。
2. **r_s 值域怪**：`r_s = 2P_S(x) − x`，support 外 = −x（可负）、support 内可正可负。UNet 吃这种带负/震荡输入（平时喂图像），行为未知。tanh 输出层（可负）起步；必要时对 r_s 做归一化。看图调整。
3. **全图 loss 可能分摊中心梯度**：组2/unet_raar 的全图 loss 让 UNet 也拟合 support 外背景振幅，support 内梯度被分摊 → 中心质量可能下降。看图确认；若中心变差，说明全图 loss 有代价，需权衡。
4. **γ=0.8 仍是猜的**：0.7 背景斑杂/未收敛、0.9 出斜线，0.8 折中未必更好。组1/组2 现象不理想则记录，不强求（γ 非主线）。
5. **HM 对 unet_raar 的影响**：HM（Zhang-Main）是项目特有约束，文献 RAAR 不带 HM。unet_raar 沿用纯 RAAR 的 HM 流程保控制变量；若不收敛可试关 HM 排查（仅诊断，正式跑仍开 HM 保公平）。
6. **样本量 5**：统计性弱；unet_raar 现象明确则够，否则补组（改 `N_GROUPS` 续跑）。
7. **CUDA 非确定性（C6）**：unet_raar 含 UNet 训练，不可复现；5 个样本取统计。
8. **指标仅初筛（R5/R8）**：末轮 6 指标仅初筛，**以 real_space.png 看图 + 收敛曲线丝滑度为准**（C13：曲线丝滑是真收敛信号）。
