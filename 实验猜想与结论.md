# 实验猜想与结论

> 本文档沉淀 `迭代模型/对比` 项目实验至今形成的猜想与结论，**严格区分"已验证结论""待验证猜想""方法论反思"**，作为后续实验（三测及以后）的认识基础。实验过程见 [初测汇报.md](初测汇报.md)、[二测汇报.md](二测汇报.md)。

---

## 1. 已验证结论（有硬实验支撑）

### C1. 甲方 HIO 实现正确
真值 support 下漂亮收敛：ssim 0.95 / amp_cc 0.93 / IoU 0.77（初测 A/B 变体）。证明严格 HIO 公式、FFT/IFFT、振幅替换、HM 实现无误。

### C2. 甲方 HIO 不偷看原图（动态 shrinkwrap）时不收敛
动态 support、3000 轮：amp_cc≈0.096、Δφ≈1.57（π/2，相位随机）、IoU≈0.038（二测）。根因是 support 估计的"鸡生蛋"循环依赖（初测已析）。**但 3000 轮对 HIO 偏少，三测将给 30000 轮重新验证。**

### C3. 四个 UNet 版本相位均未恢复
无论输出层（sigmoid / 线性 / tanh）、实空间（置 0 / HIO 反馈）、振幅 loss 对象（raw×support / raw 全图）如何组合，**Δφ 全程死卡 π/2**：

| 版本 | 配置 | 轮次 | amp_cc | Δφ |
|---|---|---|---|---|
| 1 | sigmoid + 置 0 | 5000 | 0.22 | π/2 |
| 2 | sigmoid + HIO 反馈 | 2000 | 0.000（塌缩） | π/2 |
| 3 | 线性 + HIO 反馈 | 2000 | 0.01–0.09 | π/2 |
| 4 | tanh + HIO 反馈 | 2000 | ~0.001 | π/2 |

### C4. sigmoid + HIO 反馈塌缩到平凡解
sigmoid（恒正）+ HIO 反馈，2000 轮：amp_cc=0.000（精确零）、IoU=0.000，psnr/ssim 飙到 18/0.955 是暗背景虚高假象，网络输出塌成近零/平坦图。

## 2. 核心猜想（推理 + 现象，待严格验证）

### H1. 纯振幅 loss 锁不住相位（最强猜想）
- **推理**：乙方 loss = MSE(FFT(raw) 振幅, |A_orig|)，只约束振幅，**相位完全不在优化目标**。网络没有任何动机输出"相位正确"的 raw，故相位永远随机（Δφ=π/2）。
- **支撑**：C3（四个版本 Δφ 全卡 π/2）。
- **状态**：强猜想，四次实验一致支撑，但尚未排除"轮次不足""超参不对"等其他解释。

### H2. HIO 反馈要求 ρ′ 可负振荡
- **推理**：HIO 反馈 `ρ_k−β·ρ′` 要起"阻尼清除"作用，ρ′ 需在 support 外正负振荡（正负抵消）；恒正（如 sigmoid）→ 每轮 support 外减一个固定正数 → 单调发散塌缩。
- **支撑**：C4（sigmoid+HIO 塌缩）。
- **状态**：部分验证（塌缩是事实；"必须振荡"是理论推理，未直接观测 ρ′ 的振荡性）。

### H3. 网络输出不天然振荡
- **推理**：甲方 ρ′=IFFT(全频振幅, φ) 因"强制全频反变换"在 support 外数学必然产生正负涟漪；乙方 raw=UNet 输出**没有这一步**，且网络先验偏向"平滑自然图"（背景平缓），故 support 外不振荡。即使允许负（tanh），网络也无动力画振荡。
- **支撑**：C3 第 4 版（tanh+HIO 可负仍不收敛）。
- **状态**：纯理论猜想，未直接验证。

### H4. 软学习 + HIO 反馈不兼容
- **推理**：HIO 的优势（清除背景、跳出停滞）依赖两个前提——①ρ′ 振幅每轮**精确**满足（甲方硬替换保证）；②ρ′ **天然振荡**。甲方两者都满足故 HIO > 置 0；乙方软学习两者都不满足，硬上 HIO 反而触发"必须可负 → 必须去 sigmoid → 尺度失控"的死结。
- **支撑**：C3、C4。
- **状态**：综合猜想，**正是三测要严格验证的核心**。

## 3. 方法论反思

### R1. 前期 UNet 版本对比不严格
1 版（sigmoid+置 0）跑了 5000 轮、4 版（tanh+HIO）只跑 2000 轮探测；且每版同时改动多个变量（输出层、实空间、振幅 loss 对象）。**据此下"sigmoid+置 0 比 tanh+HIO 好"的定论不严谨**——必须在同轮次、严格控制变量下重新对比。

### R2. 区分"猜想"与"结论"
前面几轮讨论中，部分判断（如"乙方缺 HIO 反馈是病根""sigmoid 是塌缩病根"）实为猜想，被后续实验部分推翻。今后：推理归猜想，严格实验验证后才能归结论。

### R3. "是否用 HIO 反馈"应作为实验目标
不预设"HIO 反馈更好/更差"，而是把"乙方是否采用 HIO 反馈（及配套输出层）"作为正经实验变量，在公平条件下对比。这正是三测的设计出发点。

## 4. 指导三测
上述猜想（尤其 H1、H4）直接定义了三测的核心问题：在严格控制变量下，对比"传统 HIO / UNet+sigmoid+置0 / UNet+tanh+HIO反馈"三种方法，让数据回答"相位是否真的锁不住""HIO 反馈在软学习里是否真的无效"。详见 [三测计划.md](三测计划.md)。
